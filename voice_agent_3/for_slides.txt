NVS:
Key Limitations and Considerations
Maximum String Length: The hard limit for a single string is 4000 bytes.
Maximum Key Length: The key (name) associated with the value is limited to a maximum of 15 characters.
Available Space: The NVS library uses a safe replacement mechanism, meaning that updating an existing entry requires space to write the new data before the old data is deleted. If your partition is heavily fragmented, you may run out of space even if the theoretical remaining capacity is sufficient.
Blob Alternative: For storing larger amounts of data, it is highly recommended to use binary large objects (blobs) instead of strings. Blob values are significantly larger, with a limit of 508,000 bytes, or 97.6% of the partition size minus 4000 bytes, whichever is lower.
